# 部署指南

## 一、环境要求

### 1.1 基础环境

| 组件 | 最低版本 | 推荐版本 | 说明 |
|------|----------|----------|------|
| JDK | 1.8 | 11/17 | 后端运行环境 |
| Node.js | 16.x | 20.x LTS | 前端构建环境 |
| Python | 3.10 | 3.12.x | AI服务运行环境 |
| MySQL | 8.0 | 8.0.x | 主数据库 |
| Redis | 6.x | 7.x | 缓存/会话存储 |
| RabbitMQ | 3.10 | 3.12.x | 消息队列 |
| Nginx | 1.20 | 1.24 | Web服务器/反向代理 |

### 1.2 服务器配置

| 配置项 | 最低配置 | 推荐配置 |
|--------|----------|----------|
| CPU | 2核 | 4核及以上 |
| 内存 | 4GB | 8GB及以上 |
| 硬盘 | 50GB | 100GB及以上 SSD |
| 带宽 | 1Mbps | 5Mbps及以上 |

## 三、环境准备

### 3.1 安装 Java (以 CentOS 为例)

```bash
# 安装 OpenJDK 17
sudo yum install -y java-17-openjdk-devel

# 验证安装
java -version

# 设置环境变量
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

### 3.2 安装 Node.js

```bash
# 使用 nvm 安装 (推荐)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20

# 验证安装
node -v
npm -v
```

### 3.3 安装 Python

```bash
# CentOS
sudo yum install -y python3.12 python3.12-devel

# 或使用 pyenv 安装
curl https://pyenv.run | bash
source ~/.bashrc
pyenv install 3.12
pyenv global 3.12

# 验证安装
python3 --version
pip3 --version
```

### 3.4 安装 MySQL 8.0

```bash
# CentOS
sudo yum localinstall -y https://dev.mysql.com/get/mysql80-community-release-el7-7.noarch.rpm
sudo yum install -y mysql-community-server

# 启动服务
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时密码并修改
sudo grep 'temporary password' /var/log/mysqld.log
mysql -u root -p

# 执行安全配置
mysql_secure_installation
```

### 3.5 安装 Redis

```bash
# CentOS
sudo yum install -y redis

# 启动服务
sudo systemctl start redis
sudo systemctl enable redis

# 配置密码 (编辑 /etc/redis.conf)
requirepass your_redis_password
```

### 3.6 安装 RabbitMQ

```bash
# CentOS - 启用 ERLANG 仓库
curl -s https://packagecloud.io/install/repositories/rabbitmq/erlang/script.rpm.sh | sudo bash
curl -s https://packagecloud.io/install/repositories/rabbitmq/rabbitmq-server/script.rpm.sh | sudo bash

# 安装 RabbitMQ
sudo yum install -y rabbitmq-server

# 启动服务
sudo systemctl start rabbitmq-server
sudo systemctl enable rabbitmq-server

# 启用管理插件
sudo rabbitmq-plugins enable rabbitmq_management

# 创建用户和权限
sudo rabbitmqctl add_user admin your_password
sudo rabbitmqctl set_user_tags admin administrator
sudo rabbitmqctl set_permissions -p / admin ".*" ".*" ".*"
```

### 3.7 安装 Nginx

```bash
# CentOS
sudo yum install -y nginx

# 启动服务
sudo systemctl start nginx
sudo systemctl enable nginx

# 配置 HTTPS (使用 Let's Encrypt)
sudo yum install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 五、后端部署

### 5.1 构建项目

```bash
# 进入后端项目目录
cd career-planning-backend

# 使用 Maven 构建
mvn clean package -DskipTests

# 构建产物
ls target/*.jar
```

### 5.2 配置应用

编辑 `src/main/resources/application-prod.yaml`:

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/career_backend?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: career
    password: your_password
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      idle-timeout: 300000
      connection-timeout: 20000
  
  redis:
    host: localhost
    port: 6379
    password: your_redis_password
    database: 0
    timeout: 10000ms

  rabbitmq:
    host: localhost
    port: 5672
    username: admin
    password: your_password
    virtual-host: /

# JWT配置
jwt:
  secret: your_jwt_secret_key_at_least_256_bits_long
  expiration: 86400000

# AI服务配置
ai:
  service:
    url: http://localhost:9000
    timeout: 30000

# 日志配置
logging:
  level:
    root: INFO
    com.backend.careerplanningbackend: INFO
  file:
    path: /var/log/career-backend
    name: /var/log/career-backend/app.log
```

### 5.3 创建启动脚本

创建 `/opt/career-backend/start.sh`:

```bash
#!/bin/bash

APP_NAME=career-planning-backend.jar
APP_HOME=/opt/career-backend
LOG_HOME=/var/log/career-backend

# 创建日志目录
mkdir -p $LOG_HOME

# 设置 JVM 参数
JAVA_OPTS="-Xms512m -Xmx2048m -XX:+UseG1GC -XX:MaxGCPauseMillis=200"

# 启动应用
cd $APP_HOME
nohup java $JAVA_OPTS -jar $APP_NAME --spring.profiles.active=prod > $LOG_HOME/console.log 2>&1 &

echo "Application started successfully!"
echo "PID: $(pgrep -f $APP_NAME)"
```

### 5.4 设置服务自启动

创建 systemd 服务文件 `/etc/systemd/system/career-backend.service`:

```ini
[Unit]
Description=Career Planning Backend Service
After=network.target mysql.service redis.service rabbitmq.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/career-backend
ExecStart=/usr/bin/java -jar /opt/career-backend/career-planning-backend.jar --spring.profiles.active=prod
Restart=on-failure
RestartSec=10
StandardOutput=append:/var/log/career-backend/stdout.log
StandardError=append:/var/log/career-backend/stderr.log

[Install]
WantedBy=multi-user.target
```

启动服务:

```bash
sudo systemctl daemon-reload
sudo systemctl enable career-backend
sudo systemctl start career-backend
sudo systemctl status career-backend
```

### 6.1 构建项目


### 6.2 配置 Nginx

创建 Nginx 配置文件 `/etc/nginx/conf.d/career-frontend.conf`:

```nginx
server {
    listen 80;
    server_name www.your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name www.your-domain.com;

    # SSL 配置 (Let's Encrypt)
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:50m;
    ssl_protocols TLSv1.2 TLSv1.3;

    # 前端静态资源
    root /var/www/career-frontend/dist;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/json application/xml;

    # 缓存配置
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 文件上传大小限制
    client_max_body_size 50M;

    # 日志
    access_log /var/log/nginx/career-frontend-access.log;
    error_log /var/log/nginx/career-frontend-error.log;
}
```

### 6.3 部署静态文件

```bash

sudo cp -r dist/* /var/www/career-frontend/

# 设置权限
```

## 七、AI 服务部署

### 7.1 使用 Docker 部署 (推荐)

```dockerfile
# Dockerfile
FROM python:3.12-slim

WORKDIR /app

# 安装依赖
COPY pyproject.toml poetry.lock ./
RUN pip install poetry && \
    poetry config virtualenvs-in-project false && \
    poetry install --no-interaction --no-ansi

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 9000

# 启动服务
CMD ["poetry", "run", "uvicorn", "main:app", "--host", "0.0.0.0", "--port", "9000", "--workers", "4"]
```

构建和运行:

```bash
# 构建镜像
cd career-planning-ai
docker build -t career-ai:latest .

# 运行容器
docker run -d \
  --name career-ai \
  -p 9000:9000 \
  -v /opt/career-ai/config.yaml:/app/config.yaml \
  -v /opt/career-ai/data:/app/data \
  -v /opt/career-ai/logs:/app/logs \
  career-ai:latest
```

### 7.2 直接部署

```bash
# 进入 AI 服务目录
cd career-planning-ai

# 安装 Poetry (如果未安装)
curl -sSL https://install.python-poetry.org | python3 -

# 安装依赖
poetry install

# 配置环境
cp config.example.yaml config.yaml
# 编辑 config.yaml 设置 API Key 等

# 启动服务
poetry run uvicorn main:app --host 0.0.0.0 --port 9000 --workers 4
```

## 十、备份与恢复

### 10.1 数据库备份脚本

创建 `/opt/backup/backup-mysql.sh`:

```bash
#!/bin/bash

BACKUP_DIR=/opt/backup/mysql
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME=career_backend
DB_USER=career
DB_PASS=your_password

# 创建备份目录
mkdir -p $BACKUP_DIR

# 执行备份
mysqldump -u$DB_USER -p$DB_PASS $DB_NAME | gzip > $BACKUP_DIR/${DB_NAME}_${DATE}.sql.gz

# 保留最近7天的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

# 同步到对象存储 (可选)
# aws s3 cp $BACKUP_DIR/${DB_NAME}_${DATE}.sql.gz s3://your-bucket/backup/

echo "Backup completed: ${DB_NAME}_${DATE}.sql.gz"
```

添加定时任务:

```bash
# 编辑 crontab
crontab -e

# 添加每日凌晨2点备份
0 2 * * * /opt/backup/backup-mysql.sh
```

### 10.2 恢复数据

```bash
# 解压备份文件
gunzip career_backend_20240413_020000.sql.gz

# 恢复数据
mysql -u career -p career_backend < career_backend_20240413_020000.sql
```
