### 技术定义

**Spring Boot**：由 Pivotal 团队提供的全新框架，其设计目的是用来简化新 Spring 应用的初始搭建以及开发过程。该框架使用了特定的方式来进行配置，从而使开发人员不再需要定义样板化的配置。本项目采用 Spring Boot 3.5.9 版本。

**MyBatis-Plus**：MyBatis-Plus 是在 MyBatis 基础上的增强工具，提供了通用的 CRUD 操作、代码生成器、分页插件等功能，极大地简化了数据库操作的开发工作。本项目采用 3.5.15 版本。

**MySQL**：MySQL 是一个关系型数据库管理系统，由瑞典 MySQL AB 公司开发，属于 Oracle 旗下产品。MySQL 是最流行的关系型数据库管理系统之一，在 WEB 应用方面，MySQL 是最好的 RDBMS 应用软件之一。本项目使用 MySQL 9.6.0 版本。

**Redis**：Redis 是一款内存高速缓存数据库，全称 Remote Dictionary Server（远程数据服务），使用 C 语言编写，是一个 key-value 存储系统，支持丰富的数据类型。本项目使用 Redis 作为缓存和会话存储。

**RabbitMQ**：云消息队列 RabbitMQ 版是一款基于高可用分布式架构实现的 AMQP 0-9-1 协议的消息产品。具备高并发、分布式、灵活扩缩容等云消息服务优势。本项目用于异步消息处理和事件驱动。

**JWT**：JWT 是基于 JSON 的标准，在不同的编程语言和平台之间都可以轻松传递和解析。JWT 是无状态的，所有信息都被包含在令牌本身中，这使得服务端不需要保存任何状态信息，从而降低了服务端的负担。本项目用于用户认证。

**Vue 3**：Vue.js 是一套用于构建用户界面的渐进式 JavaScript 框架。Vue 3 采用了 Composition API，提供了更好的 TypeScript 支持和性能优化。本项目前端采用 Vue 3.5.29 版本。

**Element Plus**：基于 Vue 3 的组件库，提供了丰富的 UI 组件，方便开发者快速构建美观的 Web 应用。本项目采用 2.13.6 版本。

**ECharts**：一个使用 JavaScript 实现的开源可视化库，可以流畅地运行在 PC 和移动设备上，兼容当前绝大部分浏览器。本项目用于数据可视化展示（雷达图、柱状图等）。

**Axios**：一个基于 Promise 的 HTTP 客户端，用于浏览器和 node.js。本项目用于前端与后端的 HTTP 通信。

**阿里云 OSS**：对象存储服务（Object Storage Service），是云计算领域的一种存储服务模型，为用户提供高可靠性、高扩展性和低成本的数据存储解决方案。本项目用于文件上传与存储。

**POI-TL**：基于 Apache POI 的 Word 模板引擎，使用 `{{}}` 作为占位符，支持文本、图片、表格、图表等元素的动态替换。本项目用于 Word 格式简历导出。

**OpenHTMLToPDF**：一个基于 iText 的 HTML 转 PDF 工具，支持 CSS3 样式和现代 Web 标准。本项目用于 PDF 格式简历导出。

**Pinia**：Vue 3 的官方状态管理库，替代 Vuex，提供了更简洁的 API 和更好的 TypeScript 支持。本项目用于前端全局状态管理。

**通义千问（Qwen）**：阿里云开发的大语言模型，支持自然语言理解、代码生成、多轮对话等功能。本项目 AI 服务采用通义千问作为底层大模型。

**FastAPI**：FastAPI 是一个现代、快速（高性能）的 Python Web 框架，用于构建 API。具有快速、高效、健壮等特点。它基于 Python 类型提示，使用 Starlette 和 Pydantic 构建。本项目 AI 服务采用此框架。

**LangChain**：一个用于开发语言模型（LLM）驱动应用的框架，旨在帮助开发者更高效地构建复杂的应用，例如智能问答系统、聊天机器人、数据分析工具等。

## 总体设计

### 项目背景

在当前就业竞争日益激烈的背景下，大学生面临着职业规划不清晰、求职准备不充分、信息不对称等问题。传统的职业规划指导往往依赖于有限的学校资源，无法为每位学生提供个性化、专业化的职业规划服务。

"职引未来"系统正是在这一背景下应运而生。系统依托 AI 大模型的强大能力，整合职业测评、简历优化、面试准备、职业知识库、AI 智能问答等多维度功能，为大学生提供全方位、个性化的职业发展支持平台。

系统采用前后端分离的微服务架构，后端基于 Spring Boot 3.5.9 构建，AI 服务基于 FastAPI + Python 构建，前端采用 Vue 3 + TypeScript。通过整合多种 AI 技术和业务模块，系统旨在帮助大学生实现科学的职业规划与就业准备，提升就业竞争力。

### 业务背景

大学生群体在职业发展过程中普遍面临以下痛点：

- **职业规划模糊**：缺乏科学的职业规划方法和工具，对职业发展方向不清晰。
- **简历制作困难**：不熟悉简历撰写技巧，无法有效展示个人优势和技能。
- **面试准备不足**：缺乏系统化的面试题库和模拟训练，面试经验不足。
- **信息获取困难**：难以获取全面、准确的职业信息和行业动态。
- **个性化服务缺失**：传统职业规划指导无法满足个性化需求。

"职引未来"系统正是为了解决这些痛点而设计。通过 AI 技术赋能职业规划全流程，帮助大学生实现从自我认知、职业探索、简历制作到面试准备的全链路职业发展服务。

### 系统架构

职引未来系统分为七层架构设计，分别是：

**展示层**：展示层是系统架构中的顶层，负责处理用户界面和用户体验。包括用户直接与之交互的所有元素，如 Vue 3 单页面应用、Element Plus 组件库、ECharts 数据可视化等。展示层的设计旨在使用户能够轻松使用系统，同时提供直观的交互和信息展示。

**接入层**：接入层通常是指网络体系结构中最接近用户或客户端设备的部分。本项目使用 Nginx 作为反向代理服务器，提供用户接入网络的门户，涵盖了 SSL 终结、静态资源服务、负载均衡、请求转发等功能。接入层的设计需要考虑到用户的接入需求、安全性和网络性能的平衡。

**服务层**：服务层是系统架构中负责提供核心服务和功能的部分。这些服务可以是应用程序接口（API）、数据处理服务、身份验证服务等。本项目使用 Spring Boot 作为核心服务层，提供 RESTful API、JWT 认证、业务逻辑处理等功能。服务层的设计目标是支持系统的核心业务逻辑，并为上层提供必要的功能支持。

**运算层**：作为系统的核心计算中枢，整合了 AI 模型推理、实时数据处理和智能决策能力。该层基于通义千问 (Qwen) 大模型实现自然语言处理（如 AI 对话、简历分析、面试题生成），通过协同过滤算法完成个性化职位推荐，并集成 Python AI 服务实现智能分析能力。同时依托消息队列 (RabbitMQ) 处理异步任务流，协调简历导出、邮件通知、积分计算等任务。

**存储层**：存储层负责管理和存储系统中的数据和信息。本项目采用多存储方案：MySQL 9.6.0 作为主数据库存储业务数据（用户信息、简历数据、面试记录等），Redis 作为缓存存储（会话管理、热点数据缓存），阿里云 OSS 作为文件存储（用户头像、简历文件等），RabbitMQ 作为消息中间件。存储层的设计需考虑到数据的安全性、可扩展性和性能需求。

**数据层**：数据层负责数据的一致性、完整性和安全性。通过 MyBatis-Plus ORM 框架实现数据访问层，确保数据操作的规范性和高效性。同时通过数据验证、事务管理、索引优化等手段保障数据质量。

**运维层**：运维层是系统架构中支持和维护整个系统运行的层级。这包括了系统监控、故障排除、性能优化、更新管理等任务。运维层的设计需要确保系统能够稳定运行，并且能够快速响应和处理各种运行时的问题和挑战。

## 系统详细设计

### 系统总功能设计

系统基于多智能体协同架构思想，将复杂的职业规划任务分解为一系列定义清晰、功能独立的子任务。系统通过 RESTful API 接口提供服务，各模块独立运行、相互协作，实现了用户管理、简历管理、面试管理、职位匹配、AI 智能助手、会员支付等核心功能。

### 基本功能设计

#### 用户基本操作

- **[功能说明]** 支持用户的注册、登录、信息查询、信息修改等基本操作。提供 RESTful 风格的 API 接口。
- **[输入]** 用户名、邮箱、密码、验证码等
- **[处理]** 对应请求执行对应的操作，包括密码 BCrypt 加密、JWT Token 生成、Redis 会话管理等
- **[返回]** 返回用户操作信息（Token、用户信息等）

#### 简历基本操作

- **[功能说明]** 支持简历的创建、编辑、查看、删除、导出等基本操作。提供在线简历编辑器，支持多种简历模板。
- **[输入]** 简历标题、模板选择、各模块内容（教育经历、工作经历、项目经历、技能等）
- **[处理]** 通过 AI 服务解析简历内容并智能填充，用户编辑后保存为 JSON 格式存储，支持导出为 Word 或 PDF 格式
- **[返回]** 返回简历操作信息、导出文件流

#### 面试基本操作

- **[功能说明]** 支持面试记录的创建、查看、编辑、删除、回顾等基本操作。支持面试日历视图。
- **[输入]** 公司名称、职位、面试时间、面试类型、状态等
- **[处理]** 对应请求执行对应的操作，支持 AI 生成面试题目
- **[返回]** 返回面试操作信息

#### 职位搜索操作

- **[功能说明]** 支持职位的搜索、查看、收藏等基本操作。支持关键词搜索、城市筛选、薪资范围筛选等。
- **[输入]** 关键词、城市、薪资范围、经验要求等
- **[处理]** 根据搜索条件在数据库中检索匹配的职位，支持分页返回结果
- **[返回]** 返回职位列表及匹配度信息

#### AI 智能助手操作

## AI 服务设计

### 智能体架构

职引未来 AI 服务采用多智能体协同架构，将复杂的职业规划任务分解为多个专业智能体：

```
┌─────────────────────────────────────────────────────┐
│                  AI 网关层                            │
│            (FastAPI + LangChain Router)             │
└─────────────────────────┬───────────────────────────┘
                          │
        ┌─────────────────┼─────────────────┐
        ▼                 ▼                 ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ 职业规划师   │  │ 简历优化师   │  │ 面试教练     │
│ CareerAgent  │  │ResumeAgent  │  │InterviewAgent│
└──────────────┘  └──────────────┘  └──────────────┘
        │                 │                 │
        └─────────────────┼─────────────────┘
                          ▼
                ┌─────────────────┐
                │   大模型服务     │
                │  (通义千问 Qwen) │
                └─────────────────┘
```

### 智能体工作流程

1. **用户请求接入**：用户通过前端发起请求，API 网关进行身份验证和请求路由
2. **意图识别**：Router Agent 识别用户意图，分发给相应的专业智能体
3. **专业处理**：专业智能体结合用户画像和历史数据进行分析和处理
4. **结果生成**：调用大模型生成最终回答，返回给用户

### 核心智能体说明

| 智能体 | 职责 | 输入 | 输出 |
|--------|------|------|------|
| RouterAgent | 意图识别与路由分发 | 用户问题 | 分发目标 |
| CareerAgent | 职业规划咨询 | 用户画像、职业目标 | 职业规划建议 |
| ResumeAgent | 简历优化建议 | 简历内容、目标岗位 | 优化建议、评分 |
| InterviewAgent | 面试辅导 | 面试岗位、面试类型 | 面试题目、答题建议 |
| AssessmentAgent | 职业能力评估 | 用户信息、测评问卷 | 能力雷达图、评估报告 |

### 环境要求

| 组件 | 要求 |
|------|------|
| JDK | OpenJDK 17+ |
| Python | Python 3.10+ |
| Node.js | Node.js 18+ |
| MySQL | MySQL 9.6.0+ |

### 后端部署

1. 安装 JDK 17 和 Maven
2. 配置 `application.yml` 数据库连接
3. 执行 `mvn clean package` 构建
4. 运行 `java -jar target/career-planning-backend.jar`

### 前端部署

1. 安装 Node.js 和 npm
2. 配置 `.env.production` 环境变量
3. 执行 `npm install` 安装依赖
4. 执行 `npm run build` 构建
5. 将 `dist` 目录部署到 Nginx

### AI 服务部署

1. 安装 Python 3.10 和 pip
2. 执行 `pip install -r requirements.txt`
3. 配置 `config.yaml` 中的 API 密钥
4. 运行 `uvicorn main:app --host 0.0.0.0 --port 9000`
