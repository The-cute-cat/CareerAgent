# Dashboard 组件单元测试

## 测试概述
为 `Dashboard.vue` 组件创建了全面的单元测试，使用 Vitest + Vue Test Utils 测试框架。

## 测试文件位置
- 测试文件: `src/views/Dashboard.test.ts`
- 配置文件: `vitest.config.ts`

## 测试覆盖范围

### 1. 组件渲染测试
- ✓ 正确渲染控制台页面
- ✓ 显示标题和欢迎语
- ✓ 显示统计卡片
- ✓ 显示快速操作区域
- ✓ 显示三个操作按钮

- ✓ 初始化统计数据

- ✓ 统计卡片有正确的类名

- ✓ 统计数据为0时正常显示

- ✓ onMounted 时加载仪表板数据

## 安装依赖

在运行测试前，需要安装以下测试依赖：

```bash
npm install -D vitest @vue/test-utils happy-dom @vitest/ui @vitest/coverage-v8
```

## 运行测试

### 运行所有测试
```bash
npm test
```

### 运行 Dashboard 测试
```bash
npm test -- Dashboard.test.ts
```

### 以监视模式运行测试
```bash
npm test -- --watch
```

### 以 UI 模式运行测试
```bash
npm run test:ui
```

### 生成覆盖率报告
```bash
npm run test:coverage
```

## 测试特性

- **模拟组件**: 使用 vi.stubGlobal 模拟 Element Plus 组件和图标
- **路由模拟**: 使用 createMemoryHistory 创建内存路由进行测试
- **状态管理**: 使用 Pinia 进行状态管理测试
- **异步测试**: 测试异步数据加载和路由跳转

## 故障排除

如果测试失败，请检查：
1. 所有测试依赖是否正确安装
2. Node.js 版本是否符合要求 (^20.19.0 || >=22.12.0)
3. Dashboard.vue 文件是否存在于 `src/views/` 目录
